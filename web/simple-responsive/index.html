Optimized for scanning

The lines in this text are 55 characters wide, so they are in the optimal range for ease of scanning. Most books use a similar line width. Everything is also justified so that our eyes can find the start and end of each line quickly. I even spaced out the lines a little, because damn the default is hard to read.

Note: 55 characters isn’t the objective optimum. Different people will probably have different preferences, but this is based on my opinion alone.

‘Unicode’ vs. 'ASCII'

We use Unicode. What’s more we don’t fuss with those HTML entities like some &rsquo; nonsense either (except that &, which we can’t avoid). Just gotta make sure we include the charset declaration, or our page gets mojibake’d: 

<meta charset="utf-8">

I considered fixing a justification style, like the inter-word or inter-character values for the CSS3 property text-justify. In the end, I decided that in general justifi­cation should be kept to a minimum by using short words and offering hyphen­ation for longer ones. For example, in the previous paragraph I moved the inline HTML code to its own line to avoid overjustification.

We can even use some weird symbols like kg⋅ms-1, or use lots of weird spaces like kg ms-1 (U+202F Narrow No-Break Space). We can really fine-tweak and leverage the typographic design that goes into our font.

I could have gone even further in this department and used ligatures, which are joined (usually pairs of) letters, but the returns are much diminishing. There used to be some ligatures quoted inline here, but including them causes the browser to download another font file, and that’s just no good.

HTML5 goodness

We can have a lot of cool stuff in this document, like a definition list:

HTML5
The markup language with which this page was written.
CSS3
The stylesheet language used to make this page look pretty.

This list uses the HTML5 definition list tags, along with the <dfn> tag, to create an accessible list of definitions for the confused.

We have a few HTML5-based options to optimise the readability of our text even more. We have the <wbr> element, which can equally be replaced by U+00AD Soft Hyphen, and gives the layout engine an opportunity to break a long word like pneumono­ultra­micro­scopic­silico­volcano­coniosis if it suits the word wrapping better.
