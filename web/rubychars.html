HTML5 defines some interesting new tags, not least of which being the <ruby> tag. <ruby> allows webpages to show ruby characters, which are annotations usually displayed above Asian (fullwidth) language characters, such as kanji. An example is the Japanese word for ruby characters: 振り仮名「ふりがな」 (furigana), or “hello”: 今日「こんにち」は (konnichi wa).

The unfortunate thing, as is the case with most modern HTML features, is that browser support is sparse. One case is when the annotation is identical to the base text. It serves no purpose, and can be hidden. Firefox implements this functionality, but other browsers do not. A workaround is to use the <rp> element, which stands for ruby parentheses, but can be used for any extra text which should show up in browsers without ruby character support. If we write <rt><rp>り</rp> in place of <rt>り, the symbol り will only appear as part of the hiragana spelling ふりがな when ruby characters are not supported. This works because the <rt> element is empty, which means the base symbol has no annotation. If we “turn off” ruby characters by writing some CSS, we can demonstrate the workaround: 

<style>
	.noruby rt { display: inline; font-size: 100%; }
	.noruby rp { display: inline; }
</style>

<ruby><rb>振<rb>り<rb>仮<rb>名<rp>「</rp><rt>ふ<rt><rp>り</rp><rt>が<rt>な<rp>」</rp></ruby>
<br>
<ruby class="noruby"><rb>振<rb>り<rb>仮<rb>名<rp>「</rp><rt>ふ<rt><rp>り</rp><rt>が<rt>な<rp>」</rp></ruby>
Produces the following:

振り仮名「ふりがな」
振り仮名「ふりがな」

Note that the markup is identical except the class applied to <ruby>. The markup is also quite verbose, which is a shame, but a lot of meaning is packed in. At least </rb> and </rt> can be omitted.
